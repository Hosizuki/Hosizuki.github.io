单开一篇整理常用指令。

Git

  1. 查看tag对应记录
    git log --pretty=oneline tag_name

  2. 切换到tag 只读。 commit记录也行
    git checkout tag_name

  3. 根据tag编辑
    git checkout -b new_branch tag_name

  4. 查找删除历史且已无用的大文件

!查
git verify-pack -v .git/objects/pack/pack-*.idx | sort -k 3 -n | tail -3
git rev-list --objects --all|grep  4b47
!删
git filter-branch --force --index-filter 'git rm -rf --cached --ignore-unmatch  mesh-0.pkl' --prune-empty --tag-name-filter cat -- --all
!彻底删
rm -rf .git/refs/original/
git reflog expire --expire=now --all
git gc --prune=now
git gc --aggressive --prune=now

文件操作

  1. 创建文件
    touch new.txt

  2. 删除文件（夹）
    rm filename
    rm -r dirname

  3. 修改权限
    chmod [-R] 777|700 filename|dirname
    其中加-R是递归修改，777是完全权限。参考：博客

  4. 节点间转移文件
    scp filename usrname@copytonodename:/home/usrname

  5. 解压
    tar -zxvf file.tar.gz -C dirname

  6. 压缩
    tar -zcvf file.tar.gz file

  7. 搜索
    find / -name filename 2>/dev/null
    grep -r 'word' file* -h

查看信息

空间

  1. 硬盘空间
    df -hl

  2. 当前文件夹下各个文件大小（屏蔽报错信息）
    du -h --max-depth=1 2>/dev/null

内存

  1. 内存
    cat /proc/meminfo
    或
    free -h
  1. 查看当前进程
    top
    1： 查看各逻辑cpu情况。
    F-方向键选择P=Last Used Cpu-空格：显示进程在哪个CPU上运行。
    q： 退出。

系统信息

  1. 操作系统发行版信息
    cat /etc/issue
  2. 网卡信息
    dmesg | grep -i eth
  3. 查看节点/主机名称
    cat /etc/hosts

CPU

  1. 逻辑CPU个数与CPU型号
    cat /proc/cpuinfo | grep name | cut -f2 -d: | uniq -c

  2. 物理CPU个数
    grep "physical id" /proc/cpuinfo|sort -u

  3. 每个物理CPU内核个数
    grep "cpu cores" /proc/cpuinfo|uniq

  4. 每个物理CPU上逻辑CPU个数
    grep "siblings" /proc/cpuinfo|uniq
    逻辑CPU个数是物理个数的两倍，说明开启了超线程。

  5. 每个逻辑CPU对应的物理位置
    cat /proc/cpuinfo | grep -E "physical id|processor"

编译

使用 OpenMPI 的 mpicc 编译MPI程序并运行

mpicc -show   //查看指令封装内容
mpicc -o hello hello_mpi.c
ldd hello     //查看链接库
mpirun -np 8 ./hello

设置运行节点：

mpirun -np 16 -machinefile nodelist ./cpi

或

mpirun -p4pg pgfile ./cpi

超线程（不建议使用）：

mpirun -oversubscribe -np 32 -machinefile nodelist ./cpi

nodelist写法：

node0 slots=2 max_slots=20
node1 slots=2 max_slots=20

参考：How do I control how my processes are scheduled across nodes
