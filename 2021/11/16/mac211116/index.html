Jupyter 不能运行 cdo 的解决办法

2021年11月16日 571 字 大概 2 分钟
  1. 1. 问题
  2. 2. 解决方案
  3. 3. Why?
  4. 4. 曲折的历程
  5. 5. 今天的吐槽

记一下之前在 Mac 上用别人的 python 程序踩的坑，虽然记不清了。文末有吐槽。

问题

在 Jupyter 里调用 cdo 会报错，大概是 FileNotFoundError: [Errno 2] No such file or directory。

解决方案

修改
/opt/anaconda3/envs/cmip/share/jupyter/kernels/python3/kernel.json
为

{
"argv": [
  "/opt/anaconda3/envs/cmip/bin/python",
  "-m",
  "ipykernel_launcher",
  "-f",
  "{connection_file}"
],
"display_name": "Python 3 (ipykernel)",
"language": "python",
"metadata": {
  "debugger": true
},
"env":{
  "PATH":"$PATH:/opt/anaconda3/envs/cmip/bin"
}
}

重启 Kernal。

Why?

python-cdo is just a binding, it requires cdo binary from the software tree. When you start your own kernel in jupyterhub cdo is not loaded per default. （参考网页）

曲折的历程

查了半天bin文件，都搜到了/opt/anaconda3/pkgs/cdo-1.9.10-h78eeb7e_3/bin，/opt/anaconda3/envs/cmip/conda-meta/cdo-1.9.10-h78eeb7e_3.json。最终是解决了。

我记得还有个有关并行处理的 bug， 因为 Mac 和 Linux 对子进程的管理标准不一样，需要改成单个 python 脚本把变量声明放到 main 函数里。
